# CivicFix - Plateforme Complète de Signalement Civique\n\n## 📋 Vue d'ensemble\n\nCivicFix est une plateforme Django 6.0 professionnelle et moderne pour la gestion des rapports civiques. Permettez aux citoyens de signaler les problèmes de leur communauté (infrastructure, environnement, santé, éducation, transport) et suivez leur résolution.\n\n## ✨ Fonctionnalités Principales\n\n### 1️⃣ Gestion des Utilisateurs\n- ✅ Authentification par email/mot de passe\n- ✅ Système de rôles (Admin, Modérateur, Utilisateur)\n- ✅ Profils utilisateur détaillés\n- ✅ Historique de connexion pour sécurité\n- ✅ API JWT pour accès par token\n\n### 2️⃣ Gestion des Rapports\n- ✅ CRUD complet (Créer, Lire, Modifier, Supprimer)\n- ✅ Statuts: Ouvert, En cours, Résolu, Fermé, Rejeté\n- ✅ Priorités: Basse, Moyenne, Élevée, Critique\n- ✅ 7 Catégories de rapports\n- ✅ Pièces jointes (PDF, documents, photos)\n- ✅ Système de commentaires collaboratifs\n- ✅ Système de votes (utile/non utile)\n- ✅ Historique d'audit complet\n\n### 3️⃣ Interface Admin & Dashboard\n- ✅ Tableau de bord avec métriques clés\n- ✅ Graphiques dynamiques (Chart.js)\n- ✅ Gestion des rapports et utilisateurs\n- ✅ Statistiques avancées par catégorie et statut\n- ✅ Logs d'activité utilisateur\n\n### 4️⃣ Design Moderne\n- ✅ TailwindCSS responsive\n- ✅ Design professionnel et minimaliste\n- ✅ Navigation intuitive\n- ✅ Mobile-friendly\n- ✅ Accessible (WCAG)\n\n### 5️⃣ API REST Complète\n- ✅ Endpoints pour tous les modèles\n- ✅ Authentification JWT\n- ✅ Pagination et filtrage\n- ✅ Documentation auto-générée\n\n## 🚀 Installation & Démarrage Rapide\n\n### Prérequis\n- Python 3.10+\n- pip\n- git\n\n### 1. Cloner le projet\n```bash\ngit clone <repo-url>\ncd civicfix_project\n```\n\n### 2. Créer l'environnement virtuel\n```bash\npython -m venv env\nsource env/bin/activate  # Linux/Mac\nenv\\Scripts\\activate     # Windows\n```\n\n### 3. Installer les dépendances\n```bash\npip install -r requirements.txt\n```\n\n### 4. Configurer les variables d'environnement\nCréer un fichier `.env`:\n```\nSECRET_KEY=votre-clé-secrète\nDEBUG=True\nALLOWED_HOSTS=localhost,127.0.0.1\nDB_NAME=civicfix\nDB_USER=postgres\nDB_PASSWORD=votre-mot-de-passe\nDB_HOST=localhost\nDB_PORT=5432\n```\n\n### 5. Migrations de base de données\n```bash\npython manage.py migrate\n```\n\n### 6. Créer un superutilisateur\n```bash\npython manage.py createsuperuser\n```\n\n### 7. Collecter les fichiers statiques\n```bash\npython manage.py collectstatic --noinput\n```\n\n### 8. Lancer le serveur de développement\n```bash\npython manage.py runserver\n```\n\nAccéder à: http://localhost:8000\n\n## 📁 Structure du Projet\n\n```\ncivicfix_project/\n├── apps/\n│   ├── accounts/ (configurable dans settings.py)\n\n### Email (pour notifications)\nConfigurer dans settings.py:\n```python\nEMAIL_BACKEND = 'django.core.mail.backends.smtp.EmailBackend'\nEMAIL_HOST = 'smtp.gmail.com'\nEMAIL_PORT = 587\nEMAIL_USE_TLS = True\nEMAIL_HOST_USER = 'votre-email@gmail.com'\nEMAIL_HOST_PASSWORD = 'votre-mot-de-passe'\n```\n\n## 🔐 Sécurité\n\n- ✅ CSRF Protection activée\n- ✅ Authentification JWT pour API\n- ✅ Gestion des sessions sécurisée\n- ✅ Validation des formulaires\n- ✅ Permissions par rôle (RBAC)\n- ✅ Rate limiting sur API\n- ✅ Historique d'accès et audit trail\n\n## 📊 Modèles de Données\n\n### User\n- ID (UUID)\n- Email, Username\n- Rôle (Admin, Modérateur, Utilisateur)\n- Avatar, Biographie\n- Sécurité (tentatives échouées, blocage temporaire)\n\n### Report\n- ID (UUID)\n- Titre, Description\n- Statut, Priorité, Catégorie\n- Auteur, Assigné à\n- Localisation (texte, latitude, longitude)\n- Notes de résolution\n- Compteurs (vues, commentaires, votes)\n- Timestamps\n\n### ReportComment\n- Contenu\n- Auteur\n- Visibilité (public/interne)\n- Timestamps\n\n### ReportAttachment\n- Fichier\n- Taille\n- Date d'upload\n\n## 🌐 Endpoints API Principaux\n\n### Authentification\n- `POST /accounts/api/register/` - Inscription\n- `POST /accounts/api/login/` - Connexion\n- `GET /accounts/api/profile/` - Profil utilisateur\n\n### Rapports\n- `GET /reports/api/reports/` - Lister les rapports\n- `POST /reports/api/reports/create/` - Créer\n- `GET /reports/api/reports/<id>/` - Détails\n- `PUT /reports/api/reports/<id>/update/` - Modifier\n- `DELETE /reports/api/reports/<id>/delete/` - Supprimer\n- `GET /reports/api/reports/<id>/comments/` - Commentaires\n\n### Dashboard\n- `GET /dashboard/api/stats/` - Statistiques\n- `GET /dashboard/api/chart-data/` - Données pour graphiques\n- `GET /dashboard/api/recent-reports/` - Rapports récents\n- `GET /dashboard/api/user-activity/` - Activité utilisateurs\n\n## 🧪 Tests\n\n```bash\npython manage.py test apps.accounts\npython manage.py test apps.reports\npython manage.py test apps.dashboard\n```\n\n## 📦 Déploiement Production\n\n### Avec Gunicorn & Nginx\n\n1. **Installer Gunicorn**\n```bash\npip install gunicorn\n```\n\n2. **Créer fichier de service systemd**\n```bash\nsudo nano /etc/systemd/system/civicfix.service\n```\n\n3. **Configurer Nginx**\n```nginx\nserver {\n    listen 80;\n    server_name yourdomain.com;\n    \n    location / {\n        proxy_pass http://127.0.0.1:8000;\n    }\n    \n    location /static/ {\n        alias /path/to/staticfiles/;\n    }\n    \n    location /media/ {\n        alias /path/to/media/;\n    }\n}\n```\n\n4. **Collecter les statics**\n```bash\npython manage.py collectstatic\n```\n\n5. **Démarrer Gunicorn**\n```bash\ngunicorn config.wsgi:application --bind 0.0.0.0:8000\n```\n\n## 🆘 Dépannage\n\n### Erreur de migration\n```bash\npython manage.py migrate --run-syncdb\n```\n\n###
