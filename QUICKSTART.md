# ⚡ Quick Start - CivicFix en 5 minutes\n\n## 1️⃣ Installation des dépendances (1 min)\n\n```bash\n# Activer l'environnement virtuel\nsource env/bin/activate          # Linux/Mac\nenv\\Scripts\\activate             # Windows\n\n# Installer les packages\npip install -r requirements.txt\n```\n\n## 2️⃣ Configuration de la BD (2 min)\n\n```bash\n# Créer les tables\npython manage.py makemigrations\npython manage.py migrate\n\n# Créer un admin\npython manage.py createsuperuser\n# Email: admin@example.com\n# Username: admin\n# Password: (votre mot de passe)\n```\n\n## 3️⃣ Démarrer le serveur (30 sec)\n\n```bash\npython manage.py runserver\n```\n\n## 4️⃣ Accéder à l'application (30 sec)\n\n- 🏠 **Accueil:** http://localhost:8000\n- 🔐 **Admin:** http://localhost:8000/admin\n- 📊 **Dashboard:** http://localhost:8000/dashboard\n- 📋 **Rapports:** http://localhost:8000/reports\n- 📡 **API:** http://localhost:8000/reports/api/reports/\n\n## 5️⃣ Tester l'API (optionnel)\n\n### Inscription\n```bash\ncurl -X POST http://localhost:8000/accounts/api/register/ \\\n  -H \"Content-Type: application/json\" \\\n  -d '{\n    \"email\": \"user@test.com\",\n    \"username\": \"testuser\",\n    \"password\": \"testpass123\",\n    \"password_confirm\": \"testpass123\"\n  }'\n```\n\n### Connexion\n```bash\nTOKEN=$(curl -X POST http://localhost:8000/accounts/api/login/ \\\n  -H \"Content-Type: application/json\" \\\n  -d '{\"email\": \"user@test.com\", \"password\": \"testpass123\"}' \\\n  | grep -o '\"access\":\"[^\"]*' | cut -d'\"' -f4)\n\necho \"Token: $TOKEN\"\n```\n\n### Créer un rapport\n```bash\ncurl -X POST http://localhost:8000/reports/api/reports/create/ \\\n  -H \"Authorization: Bearer $TOKEN\" \\\n  -H \"Content-Type: application/json\" \\\n  -d '{\n    \"title\": \"Nid de poule\",\n    \"description\": \"Cavité dangereuse\",\n    \"category\": \"infrastructure\",\n    \"priority\": \"high\"\n  }'\n```\n\n### Lister les rapports\n```bash\ncurl \"http://localhost:8000/reports/api/reports/\" \\\n  -H \"Authorization: Bearer $TOKEN\"\n```\n\n## 📚 Fichiers importants\n\n| Fichier | Purpose |\n|---------|----------|\n| `README.md` | Vue d'ensemble |\n| `INSTALLATION.md` | Guide complet |\n| `API_DOCUMENTATION.md` | API REST |\n| `ARCHITECTURE.md` | Architecture |\n| `.env.example` | Template env |\n\n## 🔥 Commandes courantes\n\n```bash\n# Tests\npython manage.py test\n\n# Collecte des statics\npython manage.py collectstatic --noinput\n\n# Shell interactif\npython manage.py shell\n\n# Vider la BD\npython manage.py flush\n\n# Voir les migrations\npython manage.py showmigrations\n```\n\n## 📝 Identifiants de test (après createsuperuser)\n\n```\nEmail: admin@example.com\nNom d'utilisateur: admin\nMoteur de passe: (celui que vous avez saisi)\nRôle: admin\n```\n\n## ✨ Première étape: créer un rapport\n\n1. Aller à http://localhost:8000/accounts/register/\n2. S'inscrire avec email et mot de passe\n3. Aller à http://localhost:8000/reports/create/\n4. Remplir le formulaire et soumettre\n5. Voir votre rapport sur http://localhost:8000/reports/\n\n## 🐛 Problèmes courants\n\n### \"No module named 'django'\"\n→ Activer l'environnement virtuel\n\n### \"no such table\"\n→ Exécuter `python manage.py migrate`\n\n### \"Port 8000 already in use\"\n→ Utiliser `python manage.py runserver 8001`\n\n## 📖 Prochaines lectures\n\n1. **INSTALLATION.md** - Setup complet & déploiement\n2. **API_DOCUMENTATION.md** - Endpoints API\n3. **ARCHITECTURE.md** - Design système\n4. **CONFIGURATION.md** - Variables d'env\n\n---\n\n**C'est tout!
